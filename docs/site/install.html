Install Aura on a personal iPhone

Aura ships as an Xcode project. Sign it with the free Apple ID Personal Team certificate and install on a personal iPhone for seven days at a time. No paid Apple Developer Program needed for evaluators or pilot users.

What you need

  • A Mac with macOS 14 (Sonoma) or newer.
  • Xcode 16 or newer, free from the Mac App Store.
  • An Apple ID — a free one is enough. Sign in to Xcode under Settings › Accounts.
  • An iPhone running iOS 17 or newer, connected via USB or paired over Wi-Fi.
  • About 10 minutes for the first build.

Step 1 — Clone the repo

git clone https://github.com/ShAuRyA-Noodle/aura-on-device.git
cd aura-on-device/apps/ios/Aura

Step 2 — Generate the Xcode project

The team does not commit the .xcodeproj. Regenerate it locally with XcodeGen, a one-time install. 

brew install xcodegen
xcodegen generate
open Aura.xcodeproj

Step 3 — Sign with your Personal Team

  1. Select the Aura target in the project navigator.
  2. Open the Signing & Capabilities tab.
  3. Tick Automatically manage signing.
  4. Pick your name as the Team. Xcode will label it as Personal Team.
  5. Change the bundle identifier to something unique to you, for example com.yourname.aura. The default com.galaxybrain.aura is reserved for the Galaxy Brain account.

Step 4 — Trust the certificate on the iPhone

  1. Plug your iPhone in and pick it from the Xcode device dropdown.
  2. Press the run button. The build will install but refuse to launch the first time.
  3. On the iPhone, open Settings › General › VPN & Device Management.
  4. Tap your Apple ID under Developer App, then Trust.
  5. Re-launch Aura from the home screen.

Step 5 — Grant permissions on first launch

Aura asks for HealthKit (HRV, sleep, steps), EventKit (calendar), and a Google sign-in for Gmail metadata. You may decline any of these — the relevant agent gracefully degrades. The first launch sets up the SQLCipher key inside the iOS Secure Enclave and writes a fresh memory graph to the app sandbox.

What the seven-day limit means

A free Apple ID Personal Team certificate is valid for seven days. After that, you have to plug the iPhone back into the same Mac and re-run the build. This is an Apple constraint on free certificates, not an Aura choice. For an unlimited install we recommend the TestFlight build that Galaxy Brain ships during Phase 2.

Troubleshooting

  • “Could not launch Aura” after install. You forgot Step 4. Trust the certificate in VPN & Device Management.
  • Bundle identifier already in use. Change com.galaxybrain.aura to something with your name.
  • HealthKit pairing fails. Open the Health app on the phone once before launching Aura — iOS has to initialise the HealthKit container.
  • Gmail sign-in loops. Confirm OAuth client IDs in Aura/Sources/Aura/Resources/Info.plist match your Google Cloud project.
